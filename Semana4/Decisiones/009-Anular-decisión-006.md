# Anular decisión 006 módulo de incidencias

## Context and Problem Statement

El patrón de diseño Prototype no es aplicable al caso de reporte de incidencias. Este patrón permite crear instancias concretas de un objeto, lo cual no es necesario ni aplicable al caso de crear incidencias que contendrán distinta información en sus campos, aunque compartan otros como: tipo de incidencia (camión averiado, reparto retrasado...).
